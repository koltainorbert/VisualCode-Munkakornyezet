Mi ez a rendszer?

Egy egysoros PowerShell parancs, amely teljesen felépíti a fejlesztői munkakörnyezetet bármilyen típusú projekthez — kérdés-felelet alapon, automatikusan.

Egy parancs — minden kész

Egyetlen irm ... | iex parancs a PowerShell ablakba — minden mást elintéz automatikusan.

Kérdés-felelet alapú konfiguráció

8 egyszerű kérdés — az összes értéknek van alapértelmezése, Enter-rel átugorható.

Valódi adatokkal tölt ki

Nem template-et hoz létre — minden generált fájlba beírja a te projekt adataidat.

Git + GitHub azonnal

git init, első commit, push, branch rendszer — mind automatikusan, az általad megadott repo URL-be.

Auto-mentés minden percben

VS Code-ba nyitáskor automatikusan elindul a napló watcher — percenként commit + push ha változás volt.

VS Code taskok azonnal

Ctrl+Shift+B = Push. Pull, Visszagörgetés, Browser Sync — mind el van készítve.

Hogyan készült?

A rendszer egy statikus HTML dokumentum + egy PowerShell script kombinációja. Semmilyen backend vagy szerver nem szükséges hozzá.

tasks.json — VS Code taskok

A .vscode/tasks.json tartalmazza az összes VS Code taskot. Ezeket Ctrl+Shift+P → Run Task-kal éred el, vagy shortcuttal.

Task neveShortcut / elérésMit csinál?
⬆ Push (feltöltés GitHubra) Ctrl+Shift+B git add . → commit (auto dátum) → git push. Ha nincs változás: "Nincs változás." üzenet.
⬇ Pull (frissítés GitHubról) Run Task → Pull git pull — letölti a GitHub-on lévő változtatásokat.
↩ Visszagörgetés Run Task → Visszagörgetés Listázza az utolsó 20 commitot, sorszám alapján visszaállítja a kiválasztott verzióra.
▶ Napló Watcher VS Code megnyitásakor auto-indul Percenként figyeli a git diff-et — ha változás van, commit + push automatikusan.
🌐 Browser Sync Run Task → Browser Sync Elindítja a live reload szervert. WordPress esetén proxy módban, HTML esetén static server módban.

A Push task az alapértelmezett build task — ezért működik Ctrl+Shift+B-vel. A többi task a Run Task menüben található.

naplo-watcher.ps1 — auto-commit

Ez a script fut a VS Code terminálban, amikor megnyitod a projektet. Percenként ellenőrzi a változásokat.

1

Változás detektálás

git diff --name-only HEAD és git ls-files --others — listázza a módosított és új fájlokat.

2

NAPLO.md frissítés

Ha az aktuális dátumhoz már van bejegyzés: hozzáadja az új sort. Ha nincs: új dátum fejlécet hoz létre.

3

Commit + push

git add . → git commit -m "Auto HH:mm — fájlnév" → git push. Ha nincs változás a cached diff-ben: kihagyja.

4

Ismétlés

Start-Sleep -Seconds 60 után újra elölről. Folyamatosan fut amíg a VS Code terminál él.

autosave-background.ps1 — Task Scheduler

A VS Code-tól független, háttérben futó backup script. Windows Task Schedulerben van regisztrálva, 5 percenként fut.

Mikor fut?

Bejelentkezéskor automatikusan elindul. Ezután 5 percenként, minden alkalommal amikor a Windows fut.

Mit csinál?

Ugyanaz mint a watcher, de minimalista: git add . → git diff --cached → ha van változás: commit + push.

Task neve

SDH-AutoSave-[projekt-neve] — Task Manager vagy Task Scheduler alkalmazásban látható.

Törlés

Task Scheduler alkalmazásban megtalálod. Jobb klikk → Delete. Vagy: Unregister-ScheduledTask -TaskName "SDH-AutoSave-projektnev"

Branch stratégia

Ha több fejlesztő dolgozik, vagy staging szerver is van, a wizard automatikusan létrehozza a branch rendszert.

main

Éles ág. Push ide = deploy éles szerverre. Csak staging ágból merge-lj ide.

staging

Tesztkörnyezet ág. Push ide = deploy staging szerverre. Develop ágból merge-lj ide tesztelés előtt.

develop

Fejlesztési ág. Here dolgozz nap mint nap. Ha kész a feature: merge staging-be.

Merge parancsok

develop → staging: git checkout staging && git merge develop && git push
staging → main (élesítés): git checkout main && git merge staging && git push

Egy fejlesztő esetén elegendő a main ág egyedül — a wizard alapértelmezésben így állítja be ([0] válasz a branch kérdésre).

GYIK — Gyakran ismételt kérdések

Miért kell a PowerShell futtatás engedélyezése?

Windows alapból tiltja az online letöltött scriptek futtatását (ExecutionPolicy Restricted). Az irm | iex parancs a script tartalmát string-ként futtatja, ezért nem szükséges módosítani a globális policy-t. A script magában biztonságos — forráskódja megtekinthető a GitHubon.

Mi az a UAC ablak a Task Scheduler regisztrációnál?

A Windows Task Scheduler feladatot csak emelt jogosultsággal lehet regisztrálni. A script Start-Process powershell -Verb RunAs paranccsal kéri ezt a jogot. Az UAC (felhasználói fiókvédelem) ablak egyszer jelenik meg — elfogadás után a háttér autosave task aktív.

Felülírja a meglévő fájlokat a script?

Igen — a script Set-Content-tel írja a fájlokat, ami felülírja a meglévőt. Ha már van tasks.json-od, azt lecseréli. Meglévő projektre való futtatás előtt mentsd el, amit nem szeretnél elveszíteni.

Működik offline, GitHub nélkül?

Igen. Ha a GitHub URL kérdésnél üresen hagyod (Enter), a script kihagyja a git push lépéseket. A git init, a taskok, a watcher és a Task Scheduler mind működik lokálisan is — csak a távoli backup nem fut.

Hogyan tudom leállítani / törölni a háttér auto-save taskot?

Kétféleképpen:

A) Task Scheduler GUI: Start → Task Scheduler → SDH-AutoSave-[projektnev] → jobb klikk → Delete

B) PowerShell:

Unregister-ScheduledTask -TaskName "SDH-AutoSave-projektnev" -Confirm:$false
Mi a különbség a Napló Watcher és az autosave-background között?

Napló Watcher (naplo-watcher.ps1): VS Code terminálban fut, 60 mp-enként, frissíti a NAPLO.md-t is, látható outputtal.

autosave-background (naplo-watcher-bg.ps1): Windows Task Schedulerben van, 5 mp-enként, ablak nélkül fut a háttérben, VS Code-tól független. Ha nincs megnyitva a projekt: akkor is pusholja a változásokat.

Browser Sync nem indul el — mit tegyek?

A Browser Sync Node.js-t és az npx parancsot igényli. Ha nincs telepítve:

# Node.js telepítése: https://nodejs.org
# Vagy globálisan telepíteni Browser Syncet:
npm install -g browser-sync

Ezután a Browser Sync task működni fog.

Hogyan frissítem a scriptet ha megjelenik egy új verzió?

Futtasd újra ugyanazt a parancsot egy meglévő projekten — felülírja a config fájlokat az újabb verzióval. A NAPLO.md és a projekt kódfájlaid érintetlenek maradnak.

Lehet Mac-en vagy Linuxon futtatni?

A script Windows-specifikus (Task Scheduler, Start-Process -Verb RunAs, LocalWP keresés). Mac-en és Linuxon PowerShell Core (PS7) fut, de a Task Scheduler és egyes Windows-specifikus parancsok nem. A script Mac/Linux portja tervben van, de nincs kész.

A GitHub Copilot miért olvassa a NAPLO.md-t?

A .github/copilot-instructions.md fájl utasítja a Copilotot a projekt-specifikus viselkedésre. A "munkamenet elején olvasd a NAPLO.md-t" instrukció azt jelenti, hogy minden chat session elején a Copilot tájékozódik: hol tartottál legutóbb, mi volt nyitott — így nem kell újra magyarázni a kontextust.
